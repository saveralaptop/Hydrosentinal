# 🗄️ Database & Authentication Structure

## Where is the User/Admin Database?

All user and admin data is stored in **Firebase Firestore** (Cloud database). Here's the complete structure:

---

## 📊 Firestore Database Structure

### **Collection: `users`**

This collection stores information about every user and admin.

**Document ID:** `{user_uid}` (Firebase's unique ID for each user)

**Document Structure:**
```json
{
  "uid": "xyz123abc",
  "email": "user@example.com",
  "role": "user",                    // ← Either "user" or "admin"
  "createdAt": "2026-04-29T10:00:00Z",
  "locations": ["default"],           // Locations user has access to
  "uniqueId": "USER_A1B2C3D4"        // Unique identifier
}
```

**Example User Document:**
```
users/
  ├── abc123xyz/
  │   ├── email: "user@demo.com"
  │   ├── role: "user"
  │   ├── createdAt: "2026-04-29T10:00:00Z"
  │   ├── locations: ["default"]
  │   └── uniqueId: "USER_ABC123"
  │
  ├── xyz789pqr/
  │   ├── email: "admin@demo.com"
  │   ├── role: "admin"
  │   ├── createdAt: "2026-04-29T10:00:00Z"
  │   ├── locations: []
  │   └── uniqueId: "ADMIN_XYZ789"
```

---

### **Collection: `devices`**

This collection stores all water quality monitoring devices.

**Document ID:** Auto-generated by Firestore

**Document Structure:**
```json
{
  "id": "device_doc_id",
  "name": "Water Quality Sensor 1",
  "uniqueId": "DEVICE_ABC12345",      // ← Unique ID for each device
  "location": "North Zone",            // ← Which location it's deployed in
  "status": "active",                  // ← "active" or "inactive"
  "createdAt": "2026-04-29T10:00:00Z"
}
```

**Example Devices Collection:**

## 🔐 How Authentication Works

### **Step 1: User Registers/Logs In**

1. User visits `/login`
2. Selects role: **User** or **Admin**
3. Enters email and password
4. System creates Firebase Auth account + Firestore user document

### **Step 2: User Role Verification**

```javascript
// In AuthContext.tsx
const login = async (email, password, selectedRole) => {
  // 1. Sign in with Firebase Auth
  const result = await signInWithEmailAndPassword(auth, email, password);
  
  // 2. Check role in Firestore
  const userDoc = await getDoc(doc(db, "users", result.user.uid));
  
  // 3. Verify role matches selected role
  if (userDoc.data().role !== selectedRole) {
    throw new Error("Role mismatch!");
  }
}
```

### **Step 3: Role-Based Routing**

After login:
- **User** → Redirected to `/dashboard`
- **Admin** → Redirected to `/admin`

## 📍 Location Intelligence System

### **Locations Available:**
```javascript
const LOCATIONS = [
  "North Zone",
  "South Zone",
  "East Zone",
  "West Zone",
  "Central Hub",
];
```

### **How Devices Map to Locations:**

1. **Admin adds device** with location "North Zone"
2. Device document is created in `devices` collection with `location: "North Zone"`
3. **User views dashboard** → Filters devices by location
4. **Location Intelligence** shows which devices are in each zone

## 🔄 User vs Admin Capabilities

### **User Account (`role: "user"`)**

✅ Can:
- View all devices with unique IDs
- See device status (Active/Inactive)
- Filter devices by location
- View location intelligence
- See device health metrics

❌ Cannot:
- Add/Edit/Delete devices
- Change locations
- Access admin panel

### **Admin Account (`role: "admin"`)**

✅ Can:
- Add new devices
- Assign devices to locations
- Edit device locations
- Delete devices
- View all device information

❌ Can:
- Access user dashboard features
- But this is not needed (full control over backend)
